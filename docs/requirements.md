# TaskFlow 要件定義書

## 1. プロジェクト概要

**アプリケーション名**: TaskFlow
**種別**: チーム向け TODO 管理 Web アプリケーション
**目的**: チームメンバーのタスクを一元管理し、進捗・優先度・作業負荷を可視化する

---

## 2. ユーザー種別と権限

### 2.1 ユーザーロール

| ロール | 説明 |
|--------|------|
| `admin` | 管理者。全メンバーのタスクを閲覧・編集・削除できる。管理者ダッシュボードにアクセス可能。 |
| `member` | 一般メンバー。自分のタスクのみ操作可能。管理者ダッシュボードは非表示。 |

### 2.2 アカウント管理

| ロール | 登録方法 |
|--------|---------|
| `admin` | Supabase ダッシュボードから手動作成。`profiles.role` を `admin` に設定する。セルフ登録不可。 |
| `member` | `/login` の「新規登録」フォームからセルフ登録可能。登録時に確認メールが送信される。 |

---

## 3. 機能要件

### 3.1 認証機能

- **FR-AUTH-1**: ログイン画面は URL で管理者用とメンバー用に分離されている
  - `/admin/login` — 管理者専用ログイン（新規登録なし）
  - `/login` — メンバー用ログイン + 新規登録
- **FR-AUTH-2**: Supabase Auth（メールアドレス + パスワード）で認証する
- **FR-AUTH-3**: ログイン状態は Supabase Auth SDK が JWT で自動管理する
- **FR-AUTH-4**: ログアウトするとセッションが破棄され、`/login` にリダイレクトされる（設定モーダルからも実行可能）
- **FR-AUTH-5**: ログイン失敗時はエラーメッセージをフォーム内に日本語で表示する
- **FR-AUTH-6**: 管理者ログイン時、Supabase の `profiles.role` を検証し `admin` でない場合は強制サインアウトしてエラーを表示する
- **FR-AUTH-7**: 認証済みユーザーが `/login` または `/admin/login` にアクセスすると `/` にリダイレクトされる
- **FR-AUTH-8**: 未認証ユーザーが保護ルートにアクセスすると `/login` にリダイレクトされる

### 3.2 タスク管理機能

#### 3.2.1 タスク作成

- **FR-TASK-1**: 「新規追加」ボタンからタスク作成モーダルを開ける
- **FR-TASK-2**: 以下の項目を入力できる

| フィールド | 必須 | デフォルト値 | 説明 |
|-----------|------|-------------|------|
| タイトル | 必須 | - | タスク名 |
| 説明 | 任意 | - | 詳細テキスト |
| 優先度 | 任意 | medium | high / medium / low |
| カテゴリ | 任意 | 仕事 | カテゴリオブジェクト（id・名前・色）から選択 |
| 開始日 | 必須 | 今日 | タスク開始日（YYYY-MM-DD） |
| 終了日 | 任意 | 開始日と同じ | タスク終了日（期間タスク対応） |
| 開始時刻 | 任意 | 09:00 | 終日の場合は非表示 |
| 終了時刻 | 任意 | 10:00 | 終日の場合は非表示 |
| 終日フラグ | 任意 | false | ON にすると時刻入力を非表示化 |
| ステータス | 任意 | todo | todo / inprogress / done |
| 作業強度 | 任意 | 3 | 1〜5 の数値（炎アイコンで表示） |

- **FR-TASK-3**: タスク作成成功時にトースト通知を表示する

#### 3.2.2 タスク編集

- **FR-TASK-4**: タスクカードのホバー時に編集アイコンが表示され、クリックで編集モーダルを開ける
- **FR-TASK-5**: カード自体のクリックでも編集モーダルを開ける
- **FR-TASK-6**: 編集モーダルには既存の値が事前入力された状態で表示される
- **FR-TASK-7**: 編集完了後にトースト通知を表示する

#### 3.2.3 タスク削除

- **FR-TASK-8**: タスクカードのホバー時に削除アイコンが表示される
- **FR-TASK-9**: 削除前に確認ダイアログ（ConfirmModal）を表示する
- **FR-TASK-10**: 削除完了後にトースト通知を表示する
- **FR-TASK-11**: 管理者は任意のメンバーのタスクを削除できる

#### 3.2.4 ステータス更新

- **FR-TASK-12**: タスクカードのホバー時にチェックボタンが表示され、クリックで完了/未完了を切り替える
- **FR-TASK-13**: ステータスに応じてカードの表示が変化する（完了時は打ち消し線・teal カラー）

### 3.3 タスクボード（TaskBoard）

- **FR-BOARD-1**: カレンダービュー／デイリービューをタブで切り替えられる
- **FR-BOARD-2**: テキスト検索（タイトル・説明を対象、大文字小文字を区別しない）
- **FR-BOARD-3**: ステータス・優先度のフィルタリングができる

#### 3.3.1 カレンダービュー

- **FR-CAL-1**: 月単位のカレンダーを表示する（週始まり：日曜日）
- **FR-CAL-2**: 各日付セルにその日のタスクを最大3件表示し、超過分は「+X」で省略表示する
- **FR-CAL-3**: 月を前後にナビゲートできる
- **FR-CAL-4**: 本日の日付をリングでハイライトする
- **FR-CAL-5**: タスクをドラッグ＆ドロップで別の日付に移動できる
- **FR-CAL-6**: タスクをクリックすると編集モーダルを開ける
- **FR-CAL-7**: タスクのカテゴリカラーを使って色分けする（カテゴリの `color` プロパティを使用）
- **FR-CAL-8**: 期間タスク（`endDate` が設定されているもの）は複数日にまたがって表示される

#### 3.3.2 デイリービュー

- **FR-DAY-1**: 指定した日付のタスク一覧をタイムライン形式で表示する
- **FR-DAY-2**: 日付を前後にナビゲートできる
- **FR-DAY-3**: タスクは開始時刻の昇順で表示される
- **FR-DAY-4**: 期限切れ（過去日付・未完了）のタスクには赤枠を表示する
- **FR-DAY-5**: 本日・翌日のタスクには「Soon」バッジを表示する

### 3.4 管理者ダッシュボード（admin のみ）

#### 3.4.1 サマリーカード

- **FR-ADMIN-1**: 総タスク数・平均完了率・活動中メンバー数・今週の新規タスク数をカード形式で表示する

#### 3.4.2 グラフ分析

- **FR-ADMIN-2**: メンバー別ワークロード比較を棒グラフ（recharts BarChart）で表示する
- **FR-ADMIN-3**: チーム全体の完了率をドーナツグラフ（recharts PieChart）で表示する

#### 3.4.3 メンバー進捗カード

- **FR-ADMIN-4**: 各メンバーの以下の情報をカード形式で表示する
  - 完了率（プログレスバー付き）
  - 完了・進行中・未着手タスク数
  - 今週のタスク数
  - 平均ワークロードスコア
  - 優先度別タスク数（high / medium / low）

### 3.5 カテゴリ管理

- **FR-CAT-1**: デフォルトカテゴリは「仕事・学習・その他」の3種類（各カテゴリは id・名前・色を持つ）
- **FR-CAT-2**: 設定モーダルからカテゴリを追加・名前変更・色変更・削除できる
- **FR-CAT-3**: カスタムカテゴリは localStorage に保存され、セッションをまたいで利用できる

### 3.6 テーマ・ユーザー設定

- **FR-SETTINGS-1**: 設定モーダルから表示名・メールアドレスを変更できる
- **FR-SETTINGS-2**: 設定モーダルからパスワードを変更できる（確認入力あり）
- **FR-SETTINGS-3**: 設定モーダルでダークモード ON/OFF を切り替えられる
- **FR-SETTINGS-4**: 設定モーダルで通知設定を ON/OFF できる（UI のみ）
- **FR-SETTINGS-5**: 設定モーダルで言語を選択できる（日本語 / English、UI のみ）
- **FR-SETTINGS-6**: テーマ設定はユーザーの preferences に保存され、ログインをまたいで保持される
- **FR-SETTINGS-7**: 初回アクセス時・設定未設定時はダークモードをデフォルトとする

### 3.7 通知機能（トースト）

- **FR-TOAST-1**: 操作結果をトースト通知で表示する（3秒後に自動消滅）
- **FR-TOAST-2**: 種別に応じて色分けする（成功：teal、エラー：赤、情報：スレート）

## 4. 非機能要件

### 4.1 対応環境

- **NFR-1**: モダンブラウザ（Chrome / Firefox / Edge / Safari 最新版）
- **NFR-2**: PC・タブレット・スマートフォンに対応するレスポンシブデザイン

### 4.2 パフォーマンス

- **NFR-3**: Vite による本番ビルドで recharts チャンクを分離し、初期ロードを最適化する

### 4.3 データ保存

- **NFR-4**: タスク・ユーザー・カテゴリのデータは Supabase PostgreSQL に保存される
- **NFR-5**: セッション情報は Supabase Auth SDK が自動管理する（JWT）

### 4.4 セキュリティ

- **NFR-6**: 認証は Supabase Auth（JWT）で管理し、パスワードは平文保存しない
- **NFR-7**: 管理者ログイン時にサーバーサイド（Supabase RLS）でロールを検証する
- **NFR-8**: 管理者アカウントはシステム管理者のみが作成可能（セルフ登録不可）
- **NFR-9**: Row Level Security（RLS）により member は自分のタスクのみアクセス可能

### 4.5 ビルド・デプロイ

- **NFR-10**: 外部 CDN に依存せず、全依存関係を npm パッケージとして管理する
- **NFR-11**: `npm run build` で本番用静的ファイルを生成できる

### 4.6 UI/UX

- **NFR-12**: 全 UI テキストは日本語で表示する
- **NFR-13**: CSS アニメーション（tailwindcss-animate）による画面遷移演出を行う
- **NFR-14**: ダークモードを基調としたスレートカラーパレットを使用する

## 5. 画面一覧

| URL | 画面 | アクセス条件 | 説明 |
|-----|------|-------------|------|
| `/login` | メンバーログイン画面 | 未認証 | メンバー用ログイン + 新規登録フォーム |
| `/admin/login` | 管理者ログイン画面 | 未認証 | 管理者専用ログインフォーム（登録なし） |
| `/` | タスクボード | 全ロール（認証済み） | カレンダー／デイリービューのタスク管理 |
| `/` | 管理者ダッシュボード | admin のみ（認証済み） | グラフ分析・メンバー統計（サイドバーから切替） |
| 設定モーダル | - | 全ロール（認証済み） | プロフィール・テーマ・カテゴリ設定 |

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| タスク | 作業単位。タイトル・日付・優先度・ステータス・作業強度などを持つ |
| 作業強度 | タスクの負荷を 1〜5 の数値で表したもの（炎アイコンで表示） |
| ワークロードスコア | ユーザーのタスクの作業強度合計を総タスク数で割った平均値 |
| 終日タスク | 時刻を指定しないタスク |
| 期間タスク | `endDate` が設定されたタスク。カレンダー上で複数日にまたがって判定される |
| 期限切れタスク | 過去日付かつ未完了のタスク |
| カテゴリ | `{ id, label, color }` を持つオブジェクト。タスクの分類に使用する |
