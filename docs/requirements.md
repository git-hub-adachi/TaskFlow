# TaskFlow 要件定義書

## 1. プロジェクト概要

**アプリケーション名**: TaskFlow
**種別**: チーム向け TODO 管理 Web アプリケーション
**目的**: チームメンバーのタスクを一元管理し、進捗・優先度・作業負荷を可視化する

---

## 2. ユーザー種別と権限

### 2.1 ユーザーロール

| ロール | 説明 |
|--------|------|
| `admin` | 管理者。全メンバーのタスクを閲覧・編集・削除できる。管理者ダッシュボードにアクセス可能。 |
| `member` | 一般メンバー。自分のタスクのみ操作可能。管理者ダッシュボードは非表示。 |

### 2.2 デモアカウント（初期データ）

| ユーザー名 | パスワード | ロール |
|------------|------------|--------|
| admin      | admin123   | admin  |
| alice      | pass123    | member |
| bob        | pass123    | member |

## 3. 機能要件

### 3.1 認証機能

- **FR-AUTH-1**: ユーザーはユーザー名とパスワードでログインできる
- **FR-AUTH-2**: ログイン状態はブラウザセッション（localStorage）に保持される
- **FR-AUTH-3**: ログアウトするとセッションが破棄され、ログイン画面に戻る
- **FR-AUTH-4**: ログイン失敗時はエラートースト通知を表示する
- **FR-AUTH-5**: ログイン成功後、admin は管理者ダッシュボード、member はデイビューへ遷移する

### 3.2 タスク管理機能

#### 3.2.1 タスク作成

- **FR-TASK-1**: 「タスクを追加」ボタンからタスク作成モーダルを開ける
- **FR-TASK-2**: 以下の項目を入力できる

| フィールド | 必須 | デフォルト値 | 説明 |
|-----------|------|-------------|------|
| タイトル | 必須 | - | タスク名 |
| 説明 | 任意 | - | 詳細テキスト |
| 優先度 | 任意 | medium | high / medium / low |
| カテゴリ | 任意 | 仕事 | プルダウンまたは新規入力 |
| 日付 | 必須 | 今日 | タスク実施日 |
| 開始時刻 | 任意 | 09:00 | 終日の場合は非表示 |
| 終了時刻 | 任意 | 10:00 | 終日の場合は非表示 |
| 終日フラグ | 任意 | false | ON にすると時刻入力を非表示化 |
| ステータス | 任意 | todo | todo / inprogress / done |
| 作業強度 | 任意 | 3 | 1〜5 の数値（炎アイコンで表示） |

- **FR-TASK-3**: カテゴリはモーダル内でその場で新規追加できる
- **FR-TASK-4**: タスク作成成功時にトースト通知を表示する

#### 3.2.2 タスク編集

- **FR-TASK-5**: タスクカードのホバー時に編集アイコンが表示され、クリックで編集モーダルを開ける
- **FR-TASK-6**: 編集モーダルには既存の値が事前入力された状態で表示される
- **FR-TASK-7**: 編集完了後にトースト通知を表示する

#### 3.2.3 タスク削除

- **FR-TASK-8**: タスクカードのホバー時に削除アイコンが表示される
- **FR-TASK-9**: 削除前に確認ダイアログを表示する
- **FR-TASK-10**: 削除完了後にトースト通知を表示する
- **FR-TASK-11**: 管理者は任意のメンバーのタスクを削除できる

#### 3.2.4 ステータス更新

- **FR-TASK-12**: タスクカードのチェックボックスをクリックするとステータスが循環する
  `todo → inprogress → done → todo`
- **FR-TASK-13**: ステータスに応じてカードの表示が変化する（完了時は打ち消し線、チェックマーク表示）

### 3.3 デイビュー（タスク一覧）

- **FR-DAY-1**: 指定した日付のタスク一覧を表示する
- **FR-DAY-2**: 日付を前後にナビゲートできる
- **FR-DAY-3**: 「今日」ボタンで本日の日付にジャンプできる
- **FR-DAY-4**: 以下のフィルタリングができる
  - 優先度（全て / high / medium / low）
  - カテゴリ（全カテゴリ / 個別カテゴリ）
  - ステータス（全て / todo / inprogress / done）
- **FR-DAY-5**: テキスト検索（タイトル・説明を対象、大文字小文字を区別しない）
- **FR-DAY-6**: タスクは開始時刻の昇順で表示される
- **FR-DAY-7**: 期限切れ（過去日付・未完了）のタスクには赤枠を表示する
- **FR-DAY-8**: 本日・翌日のタスクにはリボンバッジを表示する
- **FR-DAY-9**: 管理者は担当者名をタスクカードに表示する

### 3.4 カレンダービュー

- **FR-CAL-1**: 月単位のカレンダーを表示する（週始まり：日曜日）
- **FR-CAL-2**: 各日付セルにその日のタスクを最大3件表示し、超過分は「+X件...」で省略表示する
- **FR-CAL-3**: 月を前後にナビゲートできる
- **FR-CAL-4**: 「今日」ボタンで当月にジャンプし、本日の日付をハイライトする
- **FR-CAL-5**: タスクをドラッグ＆ドロップで別の日付に移動できる
- **FR-CAL-6**: 日付セルをクリックするとその日のデイビューへ遷移する
- **FR-CAL-7**: タスクをクリックすると編集モーダルを開ける
- **FR-CAL-8**: タスクの完了状態に応じて色分けする（未完了：青、完了：緑）

### 3.5 管理者ダッシュボード（admin のみ）

#### 3.5.1 メンバーステータス

- **FR-ADMIN-1**: 全メンバー（admin を除く）の統計情報をカード形式で表示する
- **FR-ADMIN-2**: 各メンバーカードに以下の情報を表示する
  - 完了率（プログレスバー付き）
  - 総タスク数
  - 今週のタスク数
  - 進行中タスク数
  - 平均作業強度
  - 優先度別タスク数（high / medium / low）
  - 作業負荷スコア（全タスクの強度合計）

#### 3.5.2 全タスクテーブル

- **FR-ADMIN-3**: 全メンバーのタスクを一覧テーブルで表示する
- **FR-ADMIN-4**: テーブルには以下を表示する：担当者・タイトル・日付・優先度・ステータス・操作ボタン
- **FR-ADMIN-5**: 管理者はテーブルからタスクを編集・削除できる

### 3.6 カテゴリ管理

- **FR-CAT-1**: デフォルトカテゴリは「仕事・個人・学習・その他」の4種類
- **FR-CAT-2**: タスク作成・編集モーダルで新規カテゴリを追加できる
- **FR-CAT-3**: カスタムカテゴリは localStorage に保存され、セッションをまたいで利用できる

### 3.7 テーマ機能

- **FR-THEME-1**: ヘッダーのアイコンでライト/ダークモードを切り替えられる
- **FR-THEME-2**: テーマ設定は localStorage に保存される
- **FR-THEME-3**: 初回アクセス時はシステムのカラーモード設定を自動検出する

### 3.8 通知機能（トースト）

- **FR-TOAST-1**: 操作結果をトースト通知で表示する（3秒後に自動消滅）
- **FR-TOAST-2**: 種別に応じて色分けする（成功：緑、エラー：赤、情報：青）

## 4. 非機能要件

### 4.1 対応環境

- **NFR-1**: モダンブラウザ（Chrome / Firefox / Edge / Safari 最新版）
- **NFR-2**: PC・タブレット・スマートフォンに対応するレスポンシブデザイン

### 4.2 パフォーマンス

- **NFR-3**: ローカルデータのみで動作し、ネットワーク通信なしで利用できる（認証・データ取得ともにローカル）

### 4.3 データ保存

- **NFR-4**: 全データはブラウザの localStorage に保存される
- **NFR-5**: データはデバイス・ブラウザをまたいで共有されない

### 4.4 セキュリティ（現状の制約）

- **NFR-6**: パスワードは現在平文で localStorage に保存される（要改善）
- **NFR-7**: ユーザー登録機能は存在せず、初期データとして定義済みのアカウントのみ使用可能

### 4.5 UI/UX

- **NFR-8**: 全 UI テキストは日本語で表示する
- **NFR-9**: タスク一覧・管理ダッシュボードでアニメーション演出を行う（Motion ライブラリ使用）
- **NFR-10**: ガラスモーフィズムデザインを基調とする

## 6. 用語定義

| 用語 | 定義 |
|------|------|
| タスク | 作業単位。タイトル・日付・優先度・ステータス・作業強度などを持つ |
| 作業強度 | タスクの負荷を 1〜5 の数値で表したもの（炎アイコンで表示） |
| 作業負荷スコア | メンバーが持つ全タスクの作業強度の合計値 |
| 終日タスク | 時刻を指定しないタスク。カレンダーで「終日」と表示される |
| 期限切れタスク | 過去日付かつ未完了のタスク |
